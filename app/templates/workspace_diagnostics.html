Backup & Recovery

Backup recommendations
  • Copy the database file to a safe location while the application is running using Python's sqlite3.Connection.backup() API (WAL-aware, no CLI required).
  • Always include the {{ db_file_info.display_name }}-wal and {{ db_file_info.display_name }}-shm files if you copy at the OS level — missing these may produce a corrupt backup.
  • Store backups outside the application directory and verify them periodically.
  • Test your backups by running the restore validation checks before relying on them in production.
Restore cautions
  • Stop the application and scheduler before replacing the database file.
  • After restoring, verify the backup is readable and all core tables are present before restarting.
  • Run PRAGMA integrity_check on the restored file to confirm it is intact.
  • Any operations, events, or ledger entries created after the backup was taken will be lost.
